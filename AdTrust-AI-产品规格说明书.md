# AdTrust AI 广告合规与信任审查官：产品规格说明书

## 1. 产品一句话定位

AdTrust AI 是一个面向品牌方、中小商家、内容创作者和数字营销课程学生的广告发布前 AI 审查工具，帮助用户识别“不稳妥、不可信、易误导”的营销表达，并生成更合规、更可信、更适合发布的改写方案。

产品不定位为法律咨询系统，而定位为“广告内容风险提示 + 品牌信任优化 + 发布前检查”的营销辅助工具。

核心价值：

1. 让非专业用户也能发现广告文案中的风险表达。
2. 不只指出问题，还给出可直接替换的改写文本。
3. 把广告审查从“关键词检查”升级为“语义风险 + 品牌信任”的综合分析。
4. 适合作为本科《数字营销》课程项目展示 AI 技术、营销管理和推广策略的结合。

## 2. 用户痛点分析

### 2.1 目标用户

| 用户类型 | 典型场景 | 主要目标 |
| --- | --- | --- |
| 品牌营销人员 | 广告投放、社媒发布、促销活动前审查 | 降低品牌传播风险，提高内容可信度 |
| 中小商家 | 直播话术、私域社群、商品详情页、促销海报 | 没有专业法务团队时快速自查 |
| 内容创作者 | 小红书种草、短视频口播、带货文案 | 避免夸张表达、虚假体验、广告属性不透明 |
| 营销课程学生 | 课程作业、案例分析、AI营销工具展示 | 展示数字营销、品牌信任、AI工具开发能力 |

### 2.2 核心痛点

1. 广告风险难识别：普通用户不了解绝对化表达、功效承诺、价格误导、广告属性不透明等常见问题。
2. 传统违禁词工具太粗糙：只能识别关键词，不能判断上下文，例如“第一”可能是章节编号，也可能是风险表达。
3. AI生成文案容易过度夸张：为了追求吸引力，常出现“100%有效”“3天见效”“闭眼入”等不稳妥表达。
4. 审查成本高：专业法务或品牌审查流程对中小商家、学生团队和个人创作者不现实。
5. 只避险还不够：品牌需要的不只是“少出错”，还需要让消费者觉得更真实、更专业、更可信。
6. 课程作品需要差异化：常见 AI 项目多集中在内容生成、求职、教育、电商运营，本项目用“广告合规与品牌信任审查”形成更清晰的差异点。

### 2.3 需要重点解决的风险类型

| 风险类型 | 例子 | 系统处理方式 |
| --- | --- | --- |
| 绝对化表达 | 全网第一、最好、永久有效 | 标注风险，建议弱化为“主打”“较受关注”“部分用户反馈” |
| 夸大效果 | 3天变白、一次解决、立刻见效 | 提示证据不足，改为“可能有助于”“具体因人而异” |
| 功效承诺 | 治愈、根治、包瘦、保证成绩提升 | 判定高风险，建议删除或改为非确定性描述 |
| 证据不足 | 权威认证、销量第一、专家推荐 | 要求补充数据来源、统计口径、专家身份 |
| 价格误导 | 历史最低、最后一天、全网最低价 | 提示依据不明，建议明确活动规则和时间范围 |
| 广告属性不透明 | 种草内容带购买链接但未说明合作 | 建议增加推广、合作、试用、赞助说明 |
| 用户评价滥用 | 个别案例代表普遍效果 | 提示样本不足，建议标注“个别反馈” |
| AI内容信任风险 | 虚构专家、虚构测评、虚构用户体验 | 要求避免虚构事实，保留可验证信息 |

## 3. 用户使用流程

### 3.1 主流程

1. 用户进入首页，点击“开始审查”。
2. 用户粘贴广告文本。
3. 用户选择内容类型：自动识别、小红书种草笔记、短视频脚本、直播话术、商品详情页、促销海报、私域社群文案。
4. 用户选择行业类别：自动识别、美妆护肤、食品饮料、教育培训、服饰穿搭、宠物用品、数码家电、本地生活、AI工具、其他。
5. 用户选择审查模式：快速审查或深度审查。
6. 用户选择输出偏好：保守稳妥、平衡营销效果与风险、尽量保留原文风格。
7. 系统检查输入合法性：不能为空、不能超过 5000 字、提示不要输入个人敏感信息。
8. 系统调用审查引擎，完成内容识别、风险识别、评分、改写和报告生成。
9. 用户查看综合结论、风险明细、逐句标注、改写版本和发布前检查清单。
10. 用户复制改写结果或下载 Markdown 报告。

### 3.2 异常流程

| 异常情况 | 系统处理 |
| --- | --- |
| 用户未输入文本 | 禁用提交按钮或提示“请输入需要审查的广告内容” |
| 文本超过 5000 字 | 提示用户缩短文本，或建议分段审查 |
| AI接口失败 | 返回基于规则库的简化审查结果，并提示“AI深度分析暂不可用” |
| AI返回JSON格式异常 | 后端进行二次修复或降级为规则审查结果 |
| 识别不到明显风险 | 输出低风险结论，同时提醒“低风险不等于法律合规结论” |

### 3.3 课程展示推荐流程

课堂演示时建议用三条强对比案例：

1. 美妆种草文案：展示绝对化表达、夸大效果、功效承诺。
2. 教育培训广告：展示成绩承诺、名师背书、稀缺性话术。
3. 食品代餐促销：展示瘦身承诺、0风险、明星同款、全网最低价。

演示顺序：

1. 展示原始文案。
2. 点击审查。
3. 展示风险评分和品牌信任分。
4. 展示逐句标注。
5. 展示改写前后对比。
6. 展示发布前检查清单。
7. 说明推广管理策略和有效性测试方案。

## 4. MVP功能范围

MVP目标是做出一个可运行、可截图、可讲清楚价值闭环的 Web 原型。

### 4.1 必做功能

| 模块 | 功能 | MVP验收标准 |
| --- | --- | --- |
| 首页 | 产品名称、Slogan、开始审查入口、免责声明 | 用户能理解工具用途并进入工作台 |
| 输入表单 | 广告文本、内容类型、行业、审查模式、输出偏好 | 表单可提交，有基础校验 |
| 内容识别 | 自动判断内容类型、行业、营销目的、目标受众 | 输出结构化识别结果和置信度 |
| 风险识别 | 至少识别 8 类风险 | 每个风险包含原句、类型、等级、原因、改写 |
| 风险评分 | 输出 0-100 风险分和风险等级 | 分数与风险数量、风险等级基本一致 |
| 品牌信任评分 | 输出 0-100 信任分和主要信任问题 | 能解释为什么可信度不足 |
| 逐句标注 | 按句展示风险点 | 表格清楚展示原句、风险、原因、建议 |
| 合规改写 | 生成稳妥版和营销平衡版 | 用户可直接复制 |
| 信任增强建议 | 给出 3-7 条建议 | 建议具体，例如补充证据、说明合作关系 |
| 发布前检查清单 | 输出是否通过关键检查项 | 适合放入报告或课堂展示 |
| 示例报告 | 提供 3 个内置案例 | 无需真实调用也能展示产品能力 |
| 报告复制 | 复制完整 Markdown 报告 | 用户能保存课程材料 |

### 4.2 MVP不做但需预留接口

1. 批量审查：先预留 `AuditRequest[]` 数据结构。
2. 用户账号：MVP 用 localStorage 保存本地历史。
3. 团队协作：后续再做品牌规则库、审批流。
4. 法规数据库：MVP 只做通用风险提示，不做精确法条引用。
5. 图片/视频审查：MVP 只处理文本。

## 5. 非MVP功能暂缓列表

| 功能 | 暂缓原因 | 后续价值 |
| --- | --- | --- |
| 用户登录与权限管理 | 课程项目周期有限，MVP不需要账号体系 | 支持团队版和企业版 |
| 云端历史记录 | 涉及隐私和数据库设计 | 方便复盘审查记录 |
| 批量文案审查 | 需要队列、并发和成本控制 | 面向商家和MCN有商业价值 |
| 图片海报OCR审查 | 需要OCR和多模态能力 | 可审查真实海报、商品详情图 |
| 短视频音频转写 | 需要语音识别 | 可审查直播回放和短视频口播 |
| 行业法规知识库 | 需要持续维护准确来源 | 提升专业性和说服力 |
| 平台规则适配 | 小红书、抖音、淘宝规则会变化 | 可生成平台专属建议 |
| 企业品牌语气库 | 需要用户自定义品牌规范 | 支持企业团队商业化 |
| 审批流 | 超出课程MVP复杂度 | 支持品牌发布前管理流程 |
| 数据看板 | 需要长期数据沉淀 | 展示高频风险、团队内容质量趋势 |

## 8. 智能体能力边界

### 8.1 智能体能做什么

1. 识别广告文本的内容类型、行业类别、营销目的和可能的目标受众。
2. 标注绝对化表达、夸大效果、功效承诺、证据不足、价格误导、广告属性不透明等风险。
3. 根据风险类型和严重程度生成风险评分。
4. 根据真实性、证据性、透明性、克制性、品牌一致性生成品牌信任评分。
5. 对原文逐句给出风险原因和改写建议。
6. 生成稳妥版、营销平衡版改写。
7. 生成发布前检查清单和 Markdown 审查报告。
8. 在 AI 不可用时，用本地规则库返回简化审查结果。

### 8.2 智能体不能做什么

1. 不能给出“违法/不违法”的最终法律结论。
2. 不能替代律师、平台审核员、企业法务或广告审查机构。
3. 不能保证所有风险都被识别。
4. 不能编造法规、数据来源、检测报告、专家身份或用户反馈。
5. 不能为了让文案更有吸引力而生成虚假承诺。
6. 不能保存或泄露用户输入中的敏感信息。
7. 不能将低风险结论等同于“可以绝对安全发布”。

### 8.3 输出边界话术

每份报告底部必须包含：

> 本报告仅用于营销内容风险提示、消费者信任分析和文本优化建议，不构成法律意见或平台审核结论。正式发布前，建议结合企业内部规范、平台规则和专业意见进行复核。

## 9. 信息准确性和隐私保护原则

### 9.1 信息准确性原则

1. 不编造法规条款：如果没有接入可靠法规库，不输出具体法条编号。
2. 不编造证据：如果原文没有数据来源，系统只能建议补充，不能替用户虚构。
3. 区分风险提示与事实判断：用“可能存在风险”“建议修改”，避免直接说“违法”。
4. 保留不确定性：当语境不足时，输出“需要进一步确认活动规则/数据来源/合作关系”。
5. 评分可解释：风险分和信任分必须能追溯到具体风险点。
6. 规则库与 AI 双校验：关键词命中的风险由 AI 判断上下文，AI输出再经过 JSON Schema 校验。

### 9.2 隐私保护原则

1. 输入提示：在文本框附近提示“请勿输入身份证号、手机号、客户名单、合同、未公开商业机密等敏感信息”。
2. MVP默认不上传保存历史记录；如保存，只保存在浏览器 localStorage。
3. API Key 只放服务端环境变量，前端不得出现密钥。
4. 后端日志不得记录完整用户输入，只记录请求时间、字数、模式和错误状态。
5. 导出报告只包含用户主动提交的文本和系统生成结果。
6. 后续做云端账号时，需增加删除记录、清空历史、用户授权说明。

## 10. 课堂展示亮点

### 10.1 差异化亮点

1. 不是又一个“广告文案生成器”，而是广告发布前的风险审查和信任增强工具。
2. 同时覆盖合规表达和消费者信任，能体现数字营销课程中的品牌管理思维。
3. 输出结果可解释：逐句标注、风险原因、改写建议、检查清单都能直接展示。
4. 可用真实广告样本测试，便于证明有效性。
5. 具备推广管理策略，可从校园、创作者、商家逐步扩展。

### 10.2 展示材料建议

课堂 PPT 建议包含：

1. 场景问题：AI生成营销内容变多，但审查能力跟不上。
2. 产品定位：广告发布前风险识别与品牌信任增强助手。
3. 用户画像：商家、品牌人员、创作者、营销学生。
4. 产品流程图：输入、识别、评分、标注、改写、报告。
5. 风险模型：8 类风险识别。
6. 信任模型：真实性、证据性、透明性、克制性、品牌一致性。
7. AI提示词截图：展示系统如何约束 AI 输出。
8. Web界面截图：工作台和报告页。
9. 三个测试案例：美妆、教育、食品。
10. 改写前后对比：展示风险降低和可信度提升。
11. 有效性评价：人工标注一致性、用户评分、审查效率。
12. 推广管理策略：TAM、PDCA、校园到商家的推广路径。

### 10.3 可讲的核心亮点句

本作品不追求“生成更刺激的广告”，而是解决 AI 内容爆发后更现实的问题：广告发布前如何识别风险表达、提升消费者信任，并形成可复制的内容审查流程。

### 11.4 AI提示词设计建议

System Prompt 核心约束：

```text
你是 AdTrust AI 广告合规与信任审查官。
你的任务是帮助用户在广告内容发布前识别潜在风险表达，并提出更稳妥、更可信的改写建议。

你不是律师，不提供法律结论，不判断“违法/不违法”。
你只提供营销内容风险提示、消费者信任分析、品牌传播优化建议和文本改写建议。

重点识别：
1. 绝对化表达
2. 夸大宣传
3. 功效承诺
4. 数据来源不明
5. 用户评价滥用
6. 价格促销误导
7. 广告属性不清晰
8. AI生成内容不透明
9. 容易引发消费者不信任的表达

禁止编造法规、检测报告、专家身份、用户反馈或数据来源。
必须严格按照指定 JSON Schema 输出。
```

### 11.5 开发顺序建议

第一阶段：先做静态可展示原型。

1. 首页。
2. 审查工作台。
3. 示例报告页。
4. 三个内置案例。

第二阶段：接入规则库。

1. 建立 `riskRules.json`。
2. 用关键词 + 简单上下文规则识别风险。
3. 输出简化风险报告。

第三阶段：接入 AI。

1. 新建 `/api/audit`。
2. 编写 Prompt 模板。
3. 约束 JSON Schema。
4. 前端渲染 AI 返回结果。

第四阶段：完善展示。

1. 加 loading 状态。
2. 加错误提示。
3. 加复制报告。
4. 加下载 Markdown。
5. 加 localStorage 历史记录。

第五阶段：准备课堂材料。

1. 整理 30 条测试文案。
2. 记录人工标注结果。
3. 对比 AI 识别结果。
4. 统计识别一致性、审查耗时、用户评分。

### 11.6 有效性测试建议

测试样本：30 条广告文案。

| 类型 | 数量 |
| --- | ---: |
| 小红书种草笔记 | 10 |
| 商品详情页文案 | 8 |
| 短视频脚本 | 5 |
| 直播话术 | 4 |
| 促销海报文案 | 3 |

测试指标：

| 指标 | 计算方式 |
| --- | --- |
| 风险识别数量 | 每条文案识别出的风险点数量 |
| 人工一致性 | AI识别结果与人工标注结果对比 |
| 改写可用性 | 用户评价改写是否自然、可信 |
| 信任提升 | 改写前后可信度 1-5 分对比 |
| 使用效率 | 人工审查时间与 AI审查时间对比 |

### 11.7 推广管理策略建议

第一阶段：校园场景。

面向营销课程学生、校园创业者、学生内容创作者，主打“课程作业 + 内容自查 + 案例学习”。

第二阶段：创作者场景。

面向小红书博主、抖音带货创作者、直播间主播，主打“发布前快速检查，降低踩坑风险”。

第三阶段：商家场景。

面向本地商家、电商运营、MCN机构，主打“批量审查、规则统一、品牌表达更可信”。

理论支撑：

1. TAM：强调感知有用性和感知易用性。
2. PDCA：通过样本测试、用户反馈、规则库迭代持续优化。
3. 品牌信任理论：从“避免违规”升级为“提升消费者信任”。
